# Free LLM v1.2 calibration integrity NO-GO

## Decision

The v1.2 calibration cycle is **NO-GO** with release class
`not_releasable_evidence_incomplete`.

This is an evidence-integrity decision, not a detector-performance estimate. The exact golden and
untouched runs both completed successfully over 24 repositories, and the untouched run emitted no
findings. Precision, recall, false-positive rate, TP, FP, FN, and TN are nevertheless **not
estimable**: the protocol stopped before `compute-metrics` because a required actual-run record was
not retained.

The machine-readable public decision is
[`results/v1.2-integrity-no-go.json`](../results/v1.2-integrity-no-go.json).

## Integrity failure

The golden evidence bundle canonically bound the free-core compatibility record at SHA-256
`1a18ceeed75fb97a54205dba6ca09485a4a32da131a1872600746caf323ea5fd`. The workflow created and used
that record, but neither downloaded artifact retained its exact bytes. A later reconstruction was
rejected: it is not the actual run record and does not match the bound hash.

No workflow was rerun. No evidence was repaired, replaced, reconstructed into the measurement, or
converted into manually entered counts.

## Validation

Two sequential validation passes reached the same conclusion:

1. Completeness/provenance: the required record bytes are absent from both server-digested
   artifacts, so the free-core automatic gate cannot be verified.
2. Adversarial/integrity: accepting a deterministic-looking reconstruction would weaken the
   actual-run evidence contract and permit post-result substitution.

Both passes were performed by the same AI task. They are not independent and not human reviews.

## Corrections

- The calibration workflow now copies the exact golden free-core compatibility record into the raw
  artifact before upload, so a future cycle retains the bytes bound by the evidence bundle.
- The trusted-execution verifier previously expected an `@ref` suffix in the Actions run API
  `path`. GitHub returns `.github/workflows/llm-calibration.yml` without that suffix. The verifier
  now checks the exact provider value. The live trusted-execution proof then passed for both runs,
  including ancestry, timestamps, exact workflow bytes, and server-recorded artifact digests. This
  verifier correction does not change the independent evidence-retention NO-GO.

Because the detector and evidence protocol were frozen before the untouched run, the current cycle
cannot be promoted retroactively. Any new measured release claim requires a new versioned cycle
with freshly frozen commitments and retained evidence.
