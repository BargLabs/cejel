# Free LLM v1.3 post-NO-GO detector development

## Outcome

The corrected detector matched all 65 frozen present opportunities across the 24 golden
repositories in a clean local no-egress development run. This is 100% diagnostic recall against
the old frozen present spans, up from 1/65 in the trusted v1.3 gate.

This result does not reverse the v1.3 NO-GO. Golden evidence was used to develop the detector, and
the v1.3 opportunity inventory is known to be incomplete. The result is engineering evidence for
the next detector freeze, not release evidence and not a publishable recall claim.

## Bounded corrections

The implementation adds fixture-backed support for:

- registered JavaScript and TypeScript tool-handler lineage into import-resolved filesystem and
  process effects, including executable script materialization;
- bounded same-file Python model-output lineage through local parsing helpers;
- decorated and registered Python tool inputs reaching interpreter or process sinks;
- structured model output reaching consequential Python action constructors;
- provider-neutral abstract Python model loops and explicitly unset tool-call limits;
- TypeScript and Python evaluation-result provenance, including bounded collection, return,
  persistence, and logging paths; and
- JavaScript lexical masking that preserves regular-expression literals without treating their
  contents as executable code.

The corrections retain test/example/vendor exclusions, closed-validation abstentions, deterministic
metric exclusions, finite loop/deadline/cancellation negatives, and the rule-specific evidence
contracts. The evidence threshold and release thresholds were not weakened.
