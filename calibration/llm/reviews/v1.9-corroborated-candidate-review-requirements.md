# Free LLM v1.9 corroborated-candidate review requirements

The v1.8 development-only measurement showed that deterministic corroboration across at least
three distinct query recipes yields 2,843 candidate loci. A future v1.9 cycle may use that full
set only after the following are locked before source access:

- candidate identity: repository, rule, path, start/end line, and the complete sorted supporting
  raw-hit ID set;
- a deterministic inclusion threshold of at least three distinct locked query recipes;
- no sampling, post-hoc ranking cutoff, or detector-output-driven exclusion;
- two independently attributable first-pass labels per candidate, a distinct blind auditor, and
  an adjudication record for every disagreement;
- a documented review capacity and deadline sufficient to resolve all candidates, not merely a
  selected subset;
- a fresh golden/untouched cycle if any rule, grouping behavior, evidence cap, or source set
  changes after golden source access.

This is a process requirement, not a performance claim or approval to execute untouched.
